# 03. PRD 产品需求文档

## 产品概述

产品名称：**买对衣 — 基于个人衣橱的 AI 购买决策助手**

产品目标：帮助用户在购买衣服前，基于真实衣橱、个人档案、商品截图和 AI 决策报告判断是否值得买。

核心原则：

- 不以促成购买为唯一目标。
- 不替用户做绝对审美判断。
- 不把生成图当作真实衣服事实。
- AI 输出必须可解释、可确认、可回退。

## MVP 范围

### 包含

- 邮箱注册/登录。
- 个人档案保存。
- 衣橱图片批量上传。
- 原图保存和高质量展示图生成。
- 衣服 AI 标签识别和人工确认。
- 商品截图上传。
- AI 购买决策报告。
- 决策状态按钮：决定买、先收藏、暂不考虑。
- 决策清单和状态修改。

### 不包含

- 虚拟试穿。
- 电商链接解析。
- 价格自动监控。
- 社交分享和社区。
- 完整自治 Agent。
- 自动购买或支付。
- 真人造型师服务。

## 用户流程

```text
注册/登录
-> 填写个人档案
-> 上传常穿衣服
-> AI 生成展示图和标签
-> 用户确认标签
-> 上传想买商品截图
-> AI 生成购买决策报告
-> 用户选择商品状态
-> 决策清单沉淀
```

## 功能模块

### 1. 登录与个人档案

用户目标：

- 进入自己的云端衣橱和决策记录。
- 提供基础信息，让 AI 判断更贴合场景。

字段：

- 身高、体重、BMI。
- 风格偏好。
- 不喜欢的品类。
- 常见穿搭场景。
- 预算敏感度。

规则：

- BMI 只用于版型和舒适度风险提示，不做审美评价。
- 预算敏感度用于调整价格与使用频率判断。

### 2. 衣橱录入

用户目标：

- 快速上传常穿衣服，不手动录入复杂标签。
- 获得干净、统一、适合搭配展示的衣服卡片。

核心流程：

```text
选择多张图片
-> 上传原图到 Supabase Storage
-> 创建 closet_items 记录
-> 并行调用展示图生成和衣服识别
-> 更新卡片状态
-> 用户确认或修正标签
-> 生成 embedding
```

主要状态：

- `display_image_status=queued`：展示图排队。
- `display_image_status=processing`：展示图生成中。
- `display_image_status=ready`：展示图可用。
- `display_image_status=failed`：展示图失败，可重试。
- `image_quality_flags`：记录识别状态、低置信度、背景复杂、需要确认等。

用户可编辑字段：

- 名称。
- 品类。
- 颜色。
- 版型。
- 风格标签。
- 场景标签。
- 季节标签。
- 穿着频率。

### 3. 决策聊天

用户目标：

- 上传想买商品截图，用自然语言补充意图。
- 得到一个结构化、可解释的购买建议。

输入：

- 文本问题。
- 商品截图。
- 可选价格、场景、顾虑。

输出：

- 建议状态：决定买 / 先收藏 / 暂不考虑。
- 综合总结。
- 分项分数。
- 可搭配组合。
- 重复购买风险。
- 替代单品。
- 舒适度和维护成本提示。

### 4. 决策报告

报告结构：

- 综合结论。
- 购买理由。
- 先收藏理由。
- 风险提醒。
- 搭配方案。
- 衣橱替代品。
- 长期主义判断。

报告要求：

- 必须引用真实衣橱信息。
- 推荐购买前，应至少给出可搭配逻辑。
- 不能凭空声称用户拥有不存在的衣服。
- 不能使用身体羞辱或绝对审美表达。

### 5. 决策清单

用户目标：

- 保存每次咨询过的商品。
- 回看当时的 AI 建议和自己的选择。
- 修改商品状态。

状态：

- `decided_to_buy`
- `saved_for_later`
- `not_considering`

卡片展示：

- 商品截图。
- 当前状态。
- AI 摘要。
- 搭配建议。
- 风险提醒。
- 最近咨询时间。

## 异常和失败场景

| 场景 | 处理 |
|---|---|
| 未登录访问 API | 返回 401，引导重新登录 |
| 衣服图片上传失败 | 保持本地错误提示，不创建无效记录 |
| 展示图生成失败 | 保留原图，卡片可回退原图展示，允许重试 |
| 衣服识别失败 | 标记需要人工确认，允许重试识别 |
| AI 置信度低 | 进入确认面板，不直接进入可信检索 |
| 商品截图识别失败 | 使用用户文本和 fallback 解析继续生成报告 |
| 决策模型失败 | 使用规则 fallback 报告，保证链路不中断 |
| embedding 失败 | 使用确定性 fallback embedding，保证数据可写入 |

## 权限与隐私

- 用户只能读取和修改自己的 profile、closet_items、chat_sessions、reports、decision_items。
- 衣服图片和商品截图存储在私有 Supabase Storage bucket。
- 前端通过 signed URL 访问图片。
- 文档和仓库不提交 `.env.local`、API Key、数据库连接串。

## 验收标准

- 用户能完成登录、档案保存、衣橱上传、标签确认、购买决策、加入清单的完整链路。
- 衣橱卡片优先展示 Qwen-Image-Edit 展示图，失败时回退原图。
- AI 标签未确认时，系统应提示用户确认或允许批量确认高置信度标签。
- 决策报告必须包含购买建议、搭配逻辑和风险提醒。
- 决策状态能写入并在清单中展示。
