# 05. 数据与系统链路设计

## 系统架构

```text
Next.js App Router
-> Supabase Auth
-> Supabase Storage
-> Next.js API Routes
-> AutoDL Qwen-VL / Qwen decision model
-> SiliconFlow Qwen-Image-Edit / BGE embedding
-> Supabase Postgres + pgvector
-> 前端展示与用户确认
```

## 技术选型

| 模块 | 选型 | 原因 |
|---|---|---|
| 前端 | Next.js + TypeScript + Tailwind | 快速搭建全栈 Web MVP |
| 认证 | Supabase Auth | 支持用户隔离和 RLS |
| 数据库 | Supabase Postgres | 适合结构化业务数据 |
| 图片存储 | Supabase Storage | 与用户权限和 signed URL 配合 |
| 向量检索 | pgvector | 直接在 Postgres 中存衣橱 embedding |
| AI 编排 | LangGraph.js | 适合多节点决策工作流 |
| 视觉理解 | qwen3-vl-plus | 支持衣服和商品截图识别 |
| 图片编辑 | Qwen/Qwen-Image-Edit-2509 | 生成高质量衣橱展示图 |
| Embedding | BAAI/bge-m3 | 多语言语义向量，1024 维 |

## 核心数据表

### `profiles`

保存用户基础档案：

- 身高、体重、BMI。
- 风格偏好。
- 不喜欢的品类。
- 常见场景。
- 预算敏感度。

### `closet_items`

保存用户衣橱单品：

- `image_path`：用户上传原图。
- `display_image_path`：Qwen-Image-Edit 展示图。
- `display_image_status`：展示图状态。
- `category`、`color`、`fit`、`style_tags`、`season`、`scenario_tags`。
- `embedding_text`、`embedding`。
- `image_quality_flags`、`ai_confidence`、`user_corrected`。

### `purchase_candidates`

保存用户咨询过的候选商品：

- 商品截图路径。
- 商品识别结构化信息。
- 用户购买意图。
- embedding。

### `assessment_reports`

保存 AI 决策报告：

- 决策建议。
- 分项分数。
- 购买理由、收藏理由、风险。
- 搭配组合。
- 检索上下文。

### `decision_items`

保存用户对商品的最终状态：

- 决定买。
- 先收藏。
- 暂不考虑。
- 提醒时间和复盘记录。

## 衣橱上传链路

```text
前端选择图片
-> 上传原图到 closet-images bucket
-> 插入 closet_items
-> display_image_status=queued
-> 并行调用展示图生成和衣服识别
-> 展示图写入 Storage
-> Qwen-VL 输出结构化标签
-> 用户确认标签
-> 生成 embedding
-> closet item 可用于 RAG
```

关键设计：

- 原图必须保留，因为它是事实依据。
- 展示图用于视觉体验，不替代原图。
- AI 标签默认需要确认，尤其是低置信度或复杂图片。
- 用户确认后才写入最终 embedding。

## 购买决策链路

```text
用户输入文字 + 商品截图
-> 上传截图到 purchase-screenshots bucket
-> Qwen-VL 识别商品
-> 生成候选商品 embedding
-> 检索用户衣橱
-> 检索内置穿搭知识
-> LangGraph 生成决策报告
-> 写入 assessment_reports
-> 用户选择状态
-> 写入 decision_items
```

## RAG 数据流

衣橱单品进入 RAG 的核心不是图片，而是用户确认后的 `embedding_text`。

示例：

```text
灰绿色长袖休闲衬衫，常规版型，适合春秋、通勤、日常、复古休闲，可与黑色长裤、白色T恤、深色外套搭配。
```

这样做的原因：

- 图片本身不适合直接做稳定检索。
- 用户确认后的标签比 AI 初始识别更可靠。
- 文本 embedding 更容易解释召回结果。

## 图片事实关系

| 图片类型 | 用途 | 是否作为事实来源 |
|---|---|---|
| 原图 `image_path` | 复核、识别主依据、用户查看 | 是 |
| 展示图 `display_image_path` | 衣橱卡片、搭配板、辅助识别 | 否，辅助 |
| 商品截图 `screenshot_path` | 商品识别和决策上下文 | 是，但需识别不确定性 |

## 权限设计

- 所有核心业务表开启 RLS。
- 用户只能访问自己的衣橱、聊天、商品、报告和决策清单。
- Storage 文件路径按 userId 隔离。
- 前端通过短期 signed URL 展示图片。

## 失败回退

| 失败点 | 回退策略 |
---|---|
| 展示图生成失败 | 使用原图展示，允许重试 |
| 衣服识别失败 | 标记需要人工确认，允许重试 |
| 商品识别失败 | 使用用户文本和 fallback 解析 |
| 决策模型失败 | 使用规则 fallback 报告 |
| embedding 失败 | 使用确定性 fallback embedding |
