# 07. MVP 范围与迭代路线图

## 当前 MVP 目标

MVP 目标不是做完整电商或虚拟试衣，而是验证：

> 基于个人衣橱的 AI 购买决策报告，是否能帮助用户减少冲动购买、避免重复购买，并提升衣橱利用率。

## 当前已完成能力

### 产品界面

- 登录/注册。
- 决策聊天页。
- 衣橱页。
- 决策清单页。
- 设置页。

### 数据底座

- Supabase Auth。
- Supabase Postgres。
- Supabase Storage。
- RLS 用户隔离。
- pgvector。
- 核心 migrations。

### 衣橱链路

- 批量上传衣服原图。
- 原图存储到 `closet-images`。
- Qwen-Image-Edit 生成展示图。
- Qwen-VL 原图 + 展示图双图识别。
- 标签确认面板。
- embedding 入库。

### 决策链路

- 上传商品截图。
- Qwen-VL 商品识别。
- 候选商品入库。
- 衣橱检索。
- LangGraph 决策报告。
- 报告持久化。
- 决策清单保存。

### 实验验证

- U2-Net 去背景测试：成本低，但衣架、背景残留和形变问题明显。
- Qwen-Image-Edit 测试：商品感更好，更适合衣橱卡片和搭配板。
- 结论：MVP 采用“原图 + Qwen-Image-Edit 展示图 + AI 标签 + 用户确认”。

## V1：内测版

目标：让 5-10 位目标用户完成完整链路。

重点能力：

- 稳定上传 10-30 件衣服。
- AI 标签确认流程顺畅。
- 决策报告可读、可信、可保存。
- 决策清单可回看。

评估指标：

- 衣橱上传完成率。
- 标签确认率。
- 决策报告满意度。
- 用户是否愿意再次使用。

## V2：可信 AI 版本

目标：提升 AI 输出稳定性和信任感。

重点能力：

- 展示图一致性检查。
- 原图/展示图对比评分。
- 标签修改数据回流 prompt。
- 报告引用证据更清晰。
- Rerank 接入。

评估指标：

- AI 标签修改率下降。
- 展示图重试率下降。
- 搭配建议采纳率上升。
- 重复风险识别准确率提升。

## V3：长期衣橱助手

目标：从单次购买判断扩展到长期衣橱管理。

重点能力：

- 衣橱利用率分析。
- cost-per-wear。
- 每月购物复盘。
- 风格缺口分析。
- 价格提醒。
- 已购商品穿着反馈。

评估指标：

- 用户保留率。
- 月度活跃衣橱维护次数。
- 用户减少后悔购买次数。
- 用户复穿率提升。

## V4：商品发现与生态

目标：在决策能力稳定后，再接入商品发现。

重点能力：

- 电商链接解析。
- 商品库搜索。
- 预算内替代推荐。
- 与已有衣橱相容的新品推荐。

原则：

- 不能变成单纯导购。
- 推荐商品必须解释为什么适合用户已有衣橱。
- 必须保留“暂不考虑”和“先收藏”的产品立场。

## 风险与优先级

| 风险 | 优先级 | 应对 |
|---|---|---|
| 用户不愿上传衣橱 | 高 | 降低首批上传数量，强调隐私和收益 |
| AI 图像改动衣服细节 | 高 | 保留原图，增加一致性校验 |
| AI 报告泛泛而谈 | 高 | 强制引用衣橱单品和真实场景 |
| 成本过高 | 中 | 限制批量并发和图片编辑次数 |
| 用户确认负担重 | 中 | 批量确认高置信度标签 |
| 竞品快速跟进 | 中 | 聚焦购买决策和长期主义差异化 |
